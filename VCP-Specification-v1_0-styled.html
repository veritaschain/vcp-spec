1. Introduction

1.1 Purpose

The VeritasChain Protocol (VCP) is a global standard specification for recording the "decision-making" and "execution results" of algorithmic trading in an immutable and verifiable format. VCP provides a cryptographically secured chain of evidence that establishes truth ("Veritas") in trading operations.

1.2 Scope

VCP applies to:

  • High-Frequency Trading (HFT) systems
  • Algorithmic and AI-driven trading platforms
  • Retail trading systems (MT4/MT5)
  • Cryptocurrency exchanges
  • Regulatory reporting systems

1.3 Core Modules

3. Data Model

3.1 VCP-CORE: Standard Header

Required fields for every event:

Tag Field Type Description
1001 EventID UUID Unique event identifier. MUST use UUID v7 (time-sortable) or v4
1010 Timestamp Int64 Nanoseconds since Unix epoch (UTC)
1011 EventType Int8 Event type code
1013 ClockSyncStatus Enum Clock synchronization status

3.2 Event Type Codes

CRITICAL: These codes are immutable for backward compatibility.


Trading Events (1-19):
1  = SIG    // Signal/Decision generated
2  = ORD    // Order sent
3  = ACK    // Order acknowledged
4  = EXE    // Full execution

System Events (90-109):
98 = HBT    // Heartbeat
99 = ERR    // Error
100 = REC   // Recovery
101 = SNC   // Clock sync status

3.3 Critical Precision Requirement

IMPORTANT: All financial numeric values MUST be encoded as strings in JSON to avoid IEEE 754 precision issues:


// Correct:
{ "Price": "123.456789", "Quantity": "1000.00" }

// Wrong:
{ "Price": 123.456789, "Quantity": 1000 }
